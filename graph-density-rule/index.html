Graph Density Rule

Mekanisme pengaturan kepadatan koneksi antar node dalam G-Loop agar sistem tetap stabil, tidak terlalu rapat atau terlalu renggang.

Apa itu Graph Density?

Graph Density adalah ukuran seberapa padat hubungan antar node dalam sistem G-Loop.

Terlalu sedikit koneksi → graph lemah. Terlalu banyak → graph tidak stabil.

Fungsi Graph Density Rule

  • Mengontrol jumlah koneksi antar node
  • Menjaga keseimbangan CM dan RL
  • Mencegah over-linking
  • Menghindari isolated node

Status Density

  • Low Density → node terlalu sedikit terhubung
  • Optimal Density → struktur stabil dan seimbang
  • High Density → terlalu banyak koneksi, noise meningkat

Mekanisme Kontrol

  1. Evaluasi jumlah CM (Co-Mention)
  2. Evaluasi relasi RL antar node
  3. Analisis jalur propagation
  4. Deteksi overlap koneksi
  5. Normalisasi struktur graph

Peran dalam G-Loop System

Graph Density Rule menjaga agar semua node tetap berada dalam struktur yang seimbang dan terbaca dengan jelas oleh sistem.

Graph tanpa density control akan berubah menjadi noise network.

Terhubung dengan

  • Co-Mention Network
  • Semantic Relational Graph
  • Authority Propagation Rule
  • Node Collapse Rule

Prinsip Keseimbangan

G-Loop tidak hanya tentang koneksi, tetapi tentang keseimbangan struktur koneksi tersebut.

Density Rule memastikan graph tetap bisa dibaca manusia dan mesin secara bersamaan.
