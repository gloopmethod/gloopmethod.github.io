Node Collapse Rule

Mekanisme penurunan otoritas node dalam G-Loop ketika node kehilangan koneksi, pengulangan, atau keterhubungan dalam graph.

Hub Stamp Co-Mention Relasi Propagation Collapse Density

Apa itu Node Collapse?

Node Collapse adalah kondisi ketika sebuah node kehilangan kemampuan untuk mempertahankan posisinya dalam graph G-Loop.

Tidak semua node naik. Beberapa kehilangan keterhubungan dan akhirnya menghilang dari struktur aktif.

Penyebab Collapse

  • Kehilangan Co-Mention (tidak lagi disebut)
  • Tidak ada relasi semantik baru
  • Tidak terhubung ke node kuat (anchor loss)
  • Tidak terjadi pengulangan (no reinforcement)

Jenis Collapse

  • Isolation Collapse → node terputus dari jaringan
  • Decay Collapse → otoritas turun perlahan
  • Semantic Collapse → kehilangan relevansi makna

Mekanisme Penurunan

  1. Node berhenti muncul dalam CM network
  2. Relasi RL tidak diperbarui
  3. Propagation berhenti
  4. Otoritas tidak lagi ditarik oleh graph
  5. Node masuk status “inactive”

Peran dalam G-Loop System

Node Collapse Rule menjaga keseimbangan sistem agar tidak semua node dianggap penting secara permanen.

Sistem hanya mempertahankan node yang masih memiliki keterhubungan aktif dalam graph.

Terhubung dengan

  • Authority Propagation Rule
  • Co-Mention Network
  • Semantic Relational Graph

Fungsi Keseimbangan

Collapse bukan kegagalan, tetapi mekanisme seleksi alami dalam graph G-Loop.

Tanpa collapse, semua node akan dianggap sama penting dan sistem kehilangan struktur.
